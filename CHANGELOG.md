# Changelog — novamira-adrianv2

## [Unreleased]

### Added
- **`novamira-adrianv2/elementor-check-setup`** (E1 — ROADMAP): New read-only ability in `adrianv2-elementor` category. Probes Elementor core + Pro presence, version, V3/V4 mode, active kit ID and label, global-class count, design-token count (sum of system_colors, custom_colors, system_typography, custom_typography arrays in kit settings), and current-user permissions. Returns a flat `issues[]` list for quick diagnostics. Always call before any elementor-* write ability. Registered via `includes/abilities/elementor/class-elementor-check-setup.php` + wired into `includes/abilities/elementor/bootstrap.php`.
- **`.github/workflows/ci.yml`** (I1 — ROADMAP): GitHub Actions CI with three jobs — `lint` (PHPCS), `analyze` (Psalm), `test` (PHPUnit, matrix PHP 8.0/8.1/8.2/8.3). Both lint and analyze use `|| true` to avoid blocking PRs on pre-existing warnings; flip when legacy warnings are resolved.
- **README.md — Distribution section** (H1 — ROADMAP): Documents Scenario 1 (git archive + WP-Admin ZIP upload + WP-CLI install) and Scenario 2 (composer require via `composer/installers`) with copy-paste commands for both.
- **Skill frontmatter: `title:` → `name:`** (F1 — ROADMAP): 8 skill `SKILL.md` files in `includes/skills/` had `title:` in YAML frontmatter instead of `name:`. Claude Code discovers skills by `name:` — wrong key means the skill is invisible to auto-discovery. Fixed in: `adrianv2-discover-abilities-protocol`, `adrianv2-rollback-build`, `adrianv2-self-audit`, `adrianv2-token-mapping`, `adrianv2-v3-page-edit`, `adrianv2-v3-to-v4-convert`, `adrianv2-v4-atomic-build`, `adrianv2-v4-invariants`. Only `adrianv2-live-edit` was already correct.
- **`mcp-server-config.example.json`** (K2 — ROADMAP): Example MCP server config with two entries (`novamira-adrianv2` production + `novamira-adrianv2-local` for LocalWP), connection-test command, and inline comments. New `.gitignore` prevents live `mcp-server-config.json` from being committed.

- New ability **`novamira-adrianv2/elementor-assign-class-to-containers`** (write, idempotent). Surgically assigns (or appends) a CSS class to every Elementor element whose ancestor path passes through a container selector on a single page. Routes through `Novamira\AdrianV2\Helpers\Elementor_Document_Saver::save_data` so `wp_check_post_lock` guards against overwriting a concurrent Elementor editor, and the mutation helper writes to all three slices: legacy `css_classes` + `_css_classes` and v4-atomic `settings.classes.value`. Optionally appends `custom_css` to `_elementor_page_custom_css` via `Elementor_CSS_Override::inject_page_custom_css` with the `html body …` specificity bump (0,1,0 → 0,1,2) that survives Elementor 4.x `.e-con` defaults.
- `novamira-adrianv2/update-wpcode-snippet` now accepts an OPTIONAL `bypass_kses` boolean (default `false`). When `false` the write still routes through `WPCode_Snippet::save()` (existing contract unchanged). When `true` the write routes through `Novamira\AdrianV2\Helpers\WPCode_Kses_Bypass::edit_post`, which kses-bypasses `wp_update_post` and subsequently purges WPCode's compiled-asset cache layers (options, transients, on-disk `wp-content/uploads/wpcode/cache/*.cache.php`, per-snippet `_wpcode_compiled_code` and `_wpcode_compiled_snippet` post-meta). In that mode ONLY `snippet_id`, `title`, and `code` are honoured; meta fields like `code_type`, `location`, `tags`, `priority`, `device_type`, `schedule`, `use_rules`, `rules`, `custom_shortcode`, `compress_output`, and `active` are rejected with `wpcode_bypass_kses_disallowed_fields` so the agent explicitly accepts the limited post-row semantics of the helper.
- Three production-safe helper classes under `includes/helpers/`:
  - `Novamira\AdrianV2\Helpers\WPCode_Kses_Bypass` — `edit_post(int, string, string = '', array = []): int|WP_Error` and `invalidate_compiled_cache(?int): array|WP_Error`. kses bypass is wrapped in `try/finally` so a thrown hook cannot leave later request writes unsanitised.
  - `Novamira\AdrianV2\Helpers\Elementor_Document_Saver` — `save_data(int, array): array{success: bool, warnings: string[]}` and `assign_class(array&, string, bool = true): true|WP_Error`. Returns a structured array (NOT a WP_Error union) so soft-fail warnings travel in the MCP response.
  - `Novamira\AdrianV2\Helpers\Elementor_CSS_Override` — `inject_page_custom_css(int, string): true|WP_Error` (with `html body` specificity bump and run-time `_elementor_page_settings` warning) and `generate_click_guard_script(int = 12, string = '.productslider'): string` (selectors escaped via `wp_json_encode` to prevent `</script>` XSS vectors).
- Skill **`adrianv2-live-edit`** at `includes/skills/adrianv2-live-edit/SKILL.md` in Yoast format: YAML frontmatter (`Activate when …`) + sections When to use / Domain model / Workflows / Gotchas / Conventions. Three Plan-Gotchas verbatim.
- New ability **`novamira-adrianv2/wpcode-check-setup`** (readonly, idempotent). Reports the WPCode environment health: plugin presence + version, snippet counts total/active/drafts/by code type, helper-class reachability (`Novamira\AdrianV2\Helpers\WPCode_Kses_Bypass`), presence of the three compiled-asset cache layers (`wpcode_compiled_snippets` option + `_wpcode_compiled_code`/`_wpcode_compiled_snippet` post-meta + on-disk `wp-content/uploads/wpcode/cache/`), `_wpcode_auto_demote=1` count, current-user permission sanity (`wpcode_activate_snippets` + `edit_posts`), and an `issues[]` action list. Category: `adrianv2-wpcode`. Call BEFORE `update-wpcode-snippet` with `bypass_kses=true` so the MCP client knows whether the kses bypass pathway is reachable and which cache layers need invalidating.
- New ability **`novamira-adrianv2/elementor-inject-calibrated-page`** (write, idempotent, destructive). Safe-mutation counterpart to the upstream `.claude/skills/elementor/inject.php` — routes every `_elementor_data` write through `Novamira\AdrianV2\Helpers\Elementor_Document_Saver::save_data()` so Elementor's Document API + `wp_check_post_lock` + per-post CSS rebuild all fire on the same path the Elementor editor uses on Save. Category: `adrianv2-live-edit`. Use for whole-page rebuilds after a `_clone real atoms_` workflow; for targeted class mutation use `assign-class-to-containers` instead. Inputs: `post_id` (integer ≥1) + `_elementor_data` (array) + `elementor_version` (string, soft-warns when older than active plugin) + `wp_page_template` (enum `elementor_header_footer|elementor_canvas|default`) + `mode` (enum `overwrite|merge_by_id`). Outputs: `success` + `post_id` + `sections_count` + `kit_id` (int|null from `\Elementor\Plugin::$instance->kits_manager->get_active_id()`) + soft-only `warnings[]` + `blocks_invalidated[]` (`post_css`, `files_manager_global`, `post_meta_cache`) + ISO-8601 `saved_at`. Validation gates run cheapest-first (`post_id > 0` → `class_exists(\Elementor\Plugin)` → `wp_check_post_lock` → `_elementor_data` presence → tree shape `array_is_list + first id/elType` → `check_inject_permission()` which requires only `edit_post + edit_published_pages` — `edit_theme_options` deliberately NOT required). Chicken-and-egg: on first-ever write the four boot metas (`_elementor_edit_mode=builder`, `_elementor_template_type=wp-page`, `_elementor_version`, `_wp_page_template`) are seeded via raw `update_post_meta` BEFORE delegating to `save_data()` so Elementor's `documents->get($post_id)` factory returns a `Elementor\Core\DocumentTypes\Page`; an additional `clean_post_cache($post_id)` runs between boot-meta writes and the saver so the post-meta cache bust isn't racing with the Document Loader memoisation. `merge_by_id` is recursive DFS by element `id` (top-level unmatched incoming appends at bottom; descendant recursion preserves unmatched existing leaves); `merge_by_id` with no existing tree fails loudly (`merge_no_existing_tree:` warning + `success=false`) instead of silently overwriting, so an agent's merge intent cannot be confused with overwrite intent. `meta.annotations = { readonly: false, destructive: true, idempotent: true }`. PHPUnit-Test `tests/ElementorInjectCalibratedPageTest.php` mit 19 Test-Methoden (Registration-Shape, Fail-Pfade für `invalid_post_id` / `invalid_elementor_data` / `permission_denied` / `post_locked`, Happy-Paths für overwrite + `merge_by_id` recursive-replace + `merge_by_id` append-unmatched, plus stale-version Soft-Warn). Live-Probe gegen Local-WP post_id=5373 (Elementor 4.1.3, kit_id=2839) mit allen 6 Exercise-Pfaden grün: `invalid_post_id` → fail, idempotent overwrite → success (bytes unverändert 748, `sections_count=1`, `kit_id=2839`, `blocks_invalidated_count=3`), stale-version (`elementor_version='0.0.1'`) → success mit `payload_version_old` warn, `merge_by_id` mit existing tree → success mit `sections_count=2` (matched replace + append), `merge_by_id` auf cleared page → fail (`merge_no_existing_tree`), restore-from-fixture → success (`sections_count=1`, tree top id `d4e5f6a`).
