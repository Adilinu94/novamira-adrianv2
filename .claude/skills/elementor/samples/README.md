### What was preserved verbatim

- **Element IDs** (`d4e5f6a`, `e5f6a7b`, `f6a7b8c`, `a7b8c9d`, `b8c9d0e`): 7-char alphanumeric UUIDs generated by the D1 fixture seed. They are site-internal, not production secrets, so they were preserved.
- **CSS-class tokens** (`top-level-section`, `productslider`, `mid-level`, `inner-wrap`, `product-item-title`, `add-to-cart-btn`): functional hooks that the upstream Elementor skill's `assign-class` workflow recognises. Renaming them would break recognition, so they were preserved. `productslider` is a real downstream-facing CSS class that survives the parent skill's mutator (`novamira-adrianv2/elementor-assign-class-to-containers`); keeping it lets the skill test the recursive match-by-ancestor-token path against real data.
- **Widget settings titles / texts** (`Product Title`, `Add to Cart`): generic neutral UI copy. Preserved — these are exactly the strings the upstream `inject.php` test pages use.

### What was NOT preserved

- The wrapping post-meta (`_wp_page_template = elementor_header_footer`, `_elementor_edit_mode = builder`, `_elementor_template_type = wp-page`, `_elementor_version = 4.1.3`): the sample is a `_elementor_data` JSON array only, not the full post-meta bundle. The four boot metas are reconstructed by `novamira-adrianv2/elementor-inject-calibrated-page` at injection time if the receiving page was created outside Elementor.
- The `wp_posts` row (Post ID `5373`, slug `adranv2-elementor-test`, post_title `[AdrianV2] Elementor Test`): these are site identifiers that would make the sample trivially traceable to `treets.local` in a published plugin distribution. They are deliberately NOT mirrored into the JSON. Round-tripping the file back onto any test page must use a fresh `wp_insert_post` (slug + post_title reset to non-identifying values) plus the new MCP ability's 4-seed-boot-meta path.

## Round-trip usage (the new novamira-adrianv2 MCP ability)

For downstream testing, this sample can be loaded back into any local Elementor instance as the sole `_elementor_data` payload via the `novamira-adrianv2/elementor-inject-calibrated-page` MCP ability. The ability takes the JSON contents as the `_elementor_data` parameter and:

1. validates that the payload is a non-empty list whose first entry has both `id` and `elType` keys — this sample satisfies that constraint;
2. seeds the four boot metas (`_elementor_edit_mode=builder`, `_elementor_template_type=wp-page`, `_elementor_version=4.1.3`, `_wp_page_template=elementor_header_footer`) via raw `update_post_meta` so Elementor's `documents->get($post_id)` factory returns a `Elementor\Core\DocumentTypes\Page`;
3. delegates the actual `_elementor_data` write to `Novamira\AdrianV2\Helpers\Elementor_Document_Saver::save_data()` so `wp_check_post_lock` and Elementor's `update_json_meta` cache hook fire on the same path the Elementor editor uses on Save;
4. tears down `post_css`, `files_manager_global`, and `post_meta_cache` (returned in the ability's `blocks_invalidated[]` output).

The acceptance test for the whole pipeline is: `novamira-adrianv2/elementor-inject-calibrated-page` with `post_id=<your-fresh-test-page>` + `_elementor_data=<contents-of-treets-homepage.json>` + `mode=overwrite` should yield a success response with `sections_count=1` and the four nested leaves (3 containers + 1 heading + 1 button) preserved under the same element IDs.

```bash

# 3. Copy the verified-clean payload to this directory:

  > samples/treets-homepage.json

# 5. Commit + push.
```
